# Browser Agent Integration Guide

## 🎯 Overview

The Browser Agent is a powerful tool for visual analysis of web applications generated by the platform. It uses Playwright to automatically detect CSS issues, layout problems, responsive design issues, and accessibility violations.

## ✅ Implementation Status

- ✅ **BrowserAgent Service** - Core visual analysis engine
- ✅ **BrowserPlugin** - Plugin wrapper for agent integration
- ✅ **Playwright Integration** - Headless browser automation
- ✅ **Tool Registration** - Available to all agents via PluginRegistry

## 🚀 Use Cases

### 1. **CSS & Design Problem Detection** ⭐⭐⭐
- Detects horizontal overflow
- Identifies elements outside viewport
- Finds overlapping elements (z-index issues)
- Checks for unstyled interactive elements
- Validates image loading

### 2. **Responsive Design Testing** ⭐⭐
- Tests multiple viewport sizes (desktop, tablet, mobile)
- Detects horizontal scroll at different breakpoints
- Validates touch target sizes (mobile usability)
- Identifies layout breaks

### 3. **Accessibility Auditing** ⭐
- Checks for missing alt text on images
- Validates interactive element names
- Reports accessibility violations
- Calculates accessibility score

### 4. **Performance Analysis** ⭐
- Measures page load time
- Tracks First Contentful Paint (FCP)
- Monitors Largest Contentful Paint (LCP)
- Reports DOM content loaded time

## 📦 Installation

Playwright is already installed. If you need to reinstall:

```bash
npm install playwright
npx playwright install chromium
```

## 🔌 Integration Points

### 1. **After Code Generation**
When a user generates a web app, the browser agent can automatically analyze the preview:

```typescript
// In IncrementalOrchestrator or similar
const previewUrl = await webContainerService.startDevServer();
const analysis = await browserAgent.analyzePage(previewUrl);
// Report issues to user
```

### 2. **User Request**
Users can ask Elon/Chap-ZPT to check design issues:

```
User: "Check if the design looks good"
→ Agent uses analyze_page tool
→ Reports issues and suggestions
```

### 3. **Before Deployment**
Run full audit before production deployment:

```typescript
const audit = await browserAgent.analyzePage(deploymentUrl, {
  viewports: [
    { width: 1920, height: 1080, name: 'desktop' },
    { width: 768, height: 1024, name: 'tablet' },
    { width: 375, height: 667, name: 'mobile' }
  ],
  checkAccessibility: true,
  takeScreenshot: true
});
```

## 🛠️ Available Tools

### `analyze_page`
Comprehensive visual analysis of a web page.

**Parameters:**
- `url` (required): The URL to analyze
- `viewports` (optional): Array of viewport sizes to test
- `checkAccessibility` (optional): Whether to check accessibility (default: true)

**Returns:**
```typescript
{
  url: string;
  viewport: { width: number; height: number };
  issuesFound: number;
  issues: Array<{
    type: 'layout' | 'css' | 'responsive' | 'accessibility' | 'performance' | 'visual';
    severity: 'critical' | 'high' | 'medium' | 'low';
    message: string;
    element?: string;
    suggestion?: string;
  }>;
  metrics: {
    loadTime: string;
    firstContentfulPaint?: string;
  };
  accessibility: {
    score: number;
    violations: number;
  };
  summary: string;
}
```

### `check_responsive`
Test responsive design at multiple viewport sizes.

**Parameters:**
- `url` (required): The URL to test
- `viewports` (optional): Array of viewport sizes

**Returns:**
```typescript
{
  url: string;
  viewports: Array<{
    viewport: string;
    issues: VisualIssue[];
    hasHorizontalScroll: boolean;
  }>;
  summary: string;
}
```

### `take_screenshot`
Capture a screenshot of a web page.

**Parameters:**
- `url` (required): The URL to screenshot
- `viewport` (optional): Viewport size (default: 1920x1080)
- `fullPage` (optional): Whether to capture full page (default: false)

**Returns:**
```typescript
{
  url: string;
  viewport: { width: number; height: number };
  screenshot: string; // Base64 encoded PNG
}
```

## 💡 Example Usage

### Example 1: Automatic Analysis After Generation

```typescript
// In PromptPlayground or IncrementalOrchestrator
const previewUrl = await webContainerService.startDevServer();

// Automatically analyze the preview
const analysis = await browserAgent.analyzePage(previewUrl);

if (analysis.issues.length > 0) {
  // Report issues to user
  addChatMessage({
    role: 'assistant',
    content: `Found ${analysis.issues.length} design issue(s):\n${analysis.summary}`
  });
}
```

### Example 2: User-Requested Analysis

```
User: "Can you check if the mobile layout looks good?"

Agent: Uses analyze_page tool with mobile viewport
→ Reports: "Found 2 issues: Horizontal scroll detected, touch targets too small"
```

### Example 3: Pre-Deployment Audit

```typescript
// Before deploying to production
const audit = await browserAgent.analyzePage(productionUrl, {
  viewports: [
    { width: 1920, height: 1080, name: 'desktop' },
    { width: 768, height: 1024, name: 'tablet' },
    { width: 375, height: 667, name: 'mobile' }
  ],
  checkAccessibility: true
});

if (audit.issues.some(i => i.severity === 'critical')) {
  throw new Error('Critical design issues found. Deployment blocked.');
}
```

## 🎨 Detected Issues

### Layout Issues
- Horizontal overflow
- Elements extending beyond viewport
- Overlapping elements (z-index problems)

### CSS Issues
- Unstyled interactive elements
- Missing images
- Very small text (< 12px)

### Responsive Issues
- Horizontal scroll at specific viewports
- Touch targets too small (< 44x44px on mobile)

### Accessibility Issues
- Missing alt text on images
- Interactive elements without accessible names
- Low accessibility score

## ⚙️ Configuration

The Browser Agent is automatically registered as a system plugin and doesn't require user authentication. It's available to all agents via the PluginRegistry.

## 🔧 Troubleshooting

### Browser fails to launch
- Ensure Playwright browsers are installed: `npx playwright install chromium`
- Check server has enough memory (headless browser uses ~50-100MB)

### Timeout errors
- Increase timeout in `analyzePage` options
- Check network connectivity to preview URL
- Ensure preview server is running

### Screenshot issues
- Screenshots are Base64 encoded PNG
- Large pages may take time to render
- Use `fullPage: false` for faster screenshots

## 📊 Performance

- **Analysis time**: ~2-5 seconds per viewport
- **Memory usage**: ~50-100MB per browser instance
- **Concurrent analyses**: Browser instance is reused, but only one analysis at a time
