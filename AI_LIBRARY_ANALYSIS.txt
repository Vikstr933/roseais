AI LIBRARY PLUGIN ARCHITECTURE ANALYSIS

EXECUTIVE SUMMARY
================================================================================

The AI Library codebase demonstrates a well-structured multi-agent architecture
with excellent extensibility points for building productivity integrations.

Key Findings:
- Solid BaseAgent architecture (abstract class pattern)
- Registry-based tool system for flexible capabilities
- Encrypted credential storage with key management
- Three-tier knowledge system (companies, frameworks, workspaces)
- Clear patterns for agent communication and coordination

AGENT HIERARCHY
================================================================================
BaseAgent (abstract class)
  ├── CodeGeneratorAgent
  ├── UIDesignerAgent
  ├── CompletionAgent
  ├── RequirementsAgent
  ├── ComponentArchitectAgent
  ├── StyleGeneratorAgent
  └── OrchestrationAgent (coordinates others)

All agents must implement:
  - constructor(name: string)
  - setup(): Promise<void>
  - executeTask(task: string | config): Promise<any>

TOOL REGISTRY PATTERN
================================================================================
Agents register tools during setup:

  this.toolRegistry.registerTool({
    name: 'unique-name',
    description: 'What it does',
    execute: async (...args) => { /* implementation */ }
  });

Tools can be called by any agent:
  const result = await this.toolRegistry.executeTool('tool-name', args);

Enables dynamic capability injection at runtime.

INTER-AGENT COMMUNICATION
================================================================================
SharedMemory provides key-value storage for data passing:

  // Agent 1: Store data
  sharedMemory.set('extracted-data', data);

  // Agent 2: Retrieve data
  const data = sharedMemory.require<Type>('extracted-data');

Supports:
  - Type-safe get/set
  - Namespace support
  - Snapshots for debugging
  - Existence checking

API INTEGRATION PATTERNS
================================================================================
APIKeyService handles all credential management:

- AES-256-CBC encryption for storage
- Per-request initialization vectors (IV)
- Automatic API key requirement detection
- Usage tracking (count, last_used)
- Support for: api_key, secret, token, password types

Predefined services:
  - OpenAI, Anthropic, Hugging Face
  - GitHub, Stripe

OAuth pattern (server/routes/oauth.ts):
  1. Receive OAuth callback
  2. Check/create user
  3. Create session
  4. Return session token

Database storage (db/schema-pg.ts userAPIKeys table):
  - encryptedKey (AES-256-CBC encrypted)
  - serviceName, keyName, keyType
  - lastUsed, usageCount, isActive
  - description, website

KNOWLEDGE BASE SYSTEM
================================================================================
KnowledgeService provides three-tier knowledge system:

1. Companies - product ecosystem
2. Frameworks - development frameworks
3. Workspaces - user projects

getRelevantKnowledge(prompt) process:
  1. Extract keywords and concepts
  2. Get system knowledge (top items)
  3. Search for relevant knowledge
  4. Combine: relevant > system
  5. Return top results

Knowledge API Routes:
  GET /api/knowledge/search?q=query
  GET /api/knowledge/all
  POST /api/knowledge/selected
